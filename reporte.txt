 REPORTE DE ANÁLISIS - NemoOS
===============================================================================

1. DESCRIPCIÓN GENERAL
------------------------------------------------------------------------------
NemoOS es un sistema operativo hobby para la arquitectura x86 (i386) escrito
desde cero en C++20 y Assembly (GAS/AT&T). Bootea mediante GRUB/Multiboot y
se ejecuta en QEMU. El compilador usado es GNU g++ con flags freestanding
(-ffreestanding -fno-exceptions -fno-rtti). No usa biblioteca estándar;
implementa sus propias funciones libk (memcpy, memset, strcmp, strcpy, itoa).


2. TECNOLOGÍAS UTILIZADAS
------------------------------------------------------------------------------
  Lenguaje (kernel)  : C++20 (-std=c++20, freestanding)
  Lenguaje (low-level): x86 Assembly (GAS, AT&T syntax)
  Build system       : GNU Make
  Toolchain          : g++, as, ld (x86_64-linux-gnu a i386 cruzado)
  Target             : i386 (32-bit, -m32)
  Boot protocol      : Multiboot (GRUB) / QEMU -kernel directo
  Emulador           : QEMU (qemu-system-i386)
  Linker script      : linker.ld (kernel en 1MB físico)
  Control de versión : Git

5. ESTADO DE IMPLEMENTACIÓN
------------------------------------------------------------------------------

5.1. Implementado (funcional)
  - Boot multiboot + GRUB ISO + QEMU -kernel directo
  - Terminal VGA (80x25, scroll, cursor, color, backspace)
  - Serial COM1 (38400 baud) para debug - make run-serial
  - GDT (6 entradas: null, kernel code/data, user code/data, TSS)
  - IDT (256 entradas, handlers para ISR0, IRQ0, IRQ1, page fault, syscall)
  - PIC 8259 remapeado a ints 32-47
  - Timer PIT 100Hz con contador de ticks
  - Teclado PS/2 (scancode -> ASCII, buffer circular de 256 bytes)
  - PMM (gestor de memoria física por bitmap, 0-4MB reservado)
  - Paginación: identity mapping de TODA la RAM disponible (hasta 512MB)
  - paging_map_page(), paging_map_identity() para mapeo dinámico
  - Heap con free-list: kmalloc/kfree con boundary tags, split,
    coalescing con bloques adyacentes, crecimiento automático via PMM
  - Kernel panic (mensaje + HLT)
  - Syscalls: int 0x80 con syscall 1 (get_ticks), 2 (write terminal),
    3 (read keyboard buffer)
  - Shell interactivo (help, clear, echo, about, ticks, mem, pages,
    alloc, free, kmalloc, testheap, halt, ls, touch, cat, write,
    mkdir, cd, pwd)
  - RAMFS (128 inodos, directorios, archivos, r/w)
  - Scheduler preemptivo: context switch via timer IRQ, round-robin
    entre tareas, idle cuando no hay tareas activas
  - Tareas con stack propio (4KB) y frame de interrupción inicial
  - TSS (para stack de kernel en modo usuario)
  - I/O de puertos (inb/outb/inw/outw/inl/outl)
  - libk (memcpy, memset, strcmp, strncmp, strcpy, itoa)
  - VMM: address spaces, vmm_create_space, vmm_map, vmm_unmap, vmm_switch

5.2. Parcial / Stub (incompleto)
  - ISR install: isr_install() es función vacía
  - Heap: heap.cpp/.hpp vacío (el heap real está en kmalloc.cpp)
  - VMM (Virtual Memory Manager): vmm.cpp/.hpp funcional (address spaces,
    map, unmap, switch)
  - VFS (Virtual Filesystem): vfs.cpp/.hpp vacío
  - PMM (separado): pmm.cpp/.hpp vacío (PMM real está en pmm_bitmap)
  - Scheduler: preemptivo pero sin soporte de prioridades, sin bloqueo
  - Tasks: sin PID formal, sin exit(), sin wait()

5.3. No implementado (ausente)
  - Modo usuario (Ring 3) - TSS configurado pero sin switch a user mode
  - Procesos - No hay PCB formal, PID, ni aislamiento de memoria
  - Cargador de ELF
  - Drivers de disco (ATA/IDE)
  - Sistema de archivos en disco (FAT, ext2, etc.)
  - Networking (no hay stack de red)
  - PCI (no hay enumeración de buses)
  - USB
  - IPC / señales
  - RTC/CMOS
  - Mouse PS/2
  - Syscalls: fork, exec, open, close, exit

7. CAMBIOS REALIZADOS (vs reporte anterior)
------------------------------------------------------------------------------
  - PMM: páginas reservadas de 256 a 1024 (0-4MB, cubre kernel + VGA)
  - Paging: identity mapping expandido de 4MB a toda la RAM disponible
  - Paging: nuevas API públicas paging_map_page(), paging_map_identity()
  - Heap: bump allocator reemplazado por free-list con boundary tags,
    split en kmalloc, coalescing en kfree, crecimiento vía PMM
  - Shell: comando kmalloc mejorado (escribe datos), nuevo testheap
  - ISR stub: page_fault_stub corrige error code (+4B add %esp)
  - ISR stub: irq0 modificado para llamar scheduler_tick() y hacer
    context switch (push %esp; call scheduler_tick; mov %eax, %esp)
  - Scheduler: reemplazado scheduler_run() cooperativo por
    scheduler_tick() preemptivo con round-robin y idle management
  - Task: ahora cada tarea tiene stack propio (4KB allocado via kmalloc)
    y frame de interrupción inicial para primer context switch
  - Terminal: agregado soporte serial COM1 (38400 baud)
  - Makefile: agregado target run-serial (kernel + serial stdio)
  - Makefile: targets run/debug usan -kernel (más rápido que ISO)
  - grub.cfg: agregado timeout=2
  - Bugfix crítico: scheduler_tick() tenía un camino sin return que
    causaba triple fault en el primer timer IRQ después de sti.
    Se agregó return idle_esp cuando no hay tareas activas.
  - Syscalls: agregados syscall 2 (write terminal) y 3 (read teclado)
  - Estabilidad: kmalloc ya no hace recursión infinita en OOM
  - Estabilidad: task_create con cli/sti para proteger tabla de tareas
  - Estabilidad: keyboard_buffer_push con overflow detection
  - Task: reemplazado bool active por TaskState (FREE/READY/RUNNING)
  - Task: agregado PID secuencial (next_pid)
  - Scheduler: usa TaskState en vez de active, marca RUNNING/READY
  - VMM: implementado vmm.hpp/.cpp con address spaces
  - VMM: vmm_create_space(), vmm_map(), vmm_unmap(), vmm_switch()
   - Version bump: NemoOS v0.9 -> v1.0
   - Task trampoline: nuevo task_trampoline.s en ensamblador que captura
     el retorno de la función de entrada de una tarea, la marca como DEAD
     y hace hlt con interrupciones habilitadas
   - Task: stack inicial ahora usa task_trampoline como EIP y pasa la
     dirección real de entrada vía registro EAX (slot popa)
   - Task: nueva función task_cleanup_dead() que libera pilas de tareas
     DEAD y las marca FREE
   - Scheduler: nueva función scheduler_mark_dead() para marcar tarea
     actual como DEAD desde el trampoline
   - Scheduler: scheduler_tick() ya no sobrescribe estado DEAD con READY,
     llama a task_cleanup_dead() en cada tick, salta tareas no-READY
   - Process (PCB): nuevo subsistema process/ con struct Process que unifica
     PID, estado, VMM address space, kernel stack y Task asociada
   - Process: process_create() crea PCB + VMM space + Task, los vincula
   - Process: process_destroy() mata proceso y libera VMM space
   - Process: process_switch() cambia CR3 al address space del proceso
     destino, llamado desde scheduler_tick() en cada context switch
   - Task: struct Task ahora tiene campo Process* process (back-pointer)
   - Task: task_create() retorna Task* (antes void), acepta Process* opcional
   - Task: task_cleanup_dead() limpia process = nullptr
   - Scheduler: scheduler_mark_dead() también marca Process DEAD si la
     tarea era su main_task
    - Scheduler: scheduler_tick() captura prev_idx y llama process_switch()
      al migrar entre procesos
    - Bugfix CRÍTICO: task_create() tenía orden de registros invertido en el
      stack inicial (popa espera EDI primero, EAX último; el código ponía
      EAX primero, EDI último). Causaba call *%eax = 0 → page fault.
    - Bugfix: scheduler_tick() saltaba el índice 0 de la tabla de tareas
      porque inicializaba next=0 y luego incrementaba antes de revisar,
      haciendo que la iteración 16 saliera sin revisar [0].
      Fix: usar current_task (-1) como inicial, así (0+1)%16=0 revisa [0].
    - Validación en QEMU: process_create + process_switch + VMM switch +
      context switch funcionando correctamente. Salida:
      "Process 1 created" → "Process 1 alive"
    - Keyboard: driver reescrito con soporte completo de Shift (izq/der),
      Caps Lock, y keymap US completo de 128 scancodes
    - Keyboard: corrección de múltiples entradas incorrectas en el keymap
      original (0x0C, 0x0D, 0x1A, 0x1B, 0x27-0x29, 0x35)
    - Keyboard: keymap dual lower/upper para caracteres shift + símbolos
      especiales (!@#$%^&*()_+{}:\"<>?)
    - Shell: nuevos comandos whoami, uname, uptime, yes, true, false,
      wc <file>, head <file>, rev <file>
    - Shell: help actualizado con todos los comandos
